OS

概述

OS 是系统软件

定义

操作系统（Operating System，OS）
  • 是一种软件程序
  • 提供统一接口（将底层硬件实现的各种运算功能抽象为一种统一的接口，供其他程序调用）
  • 是管理计算机硬件与软件资源的计算机程序
  • 同时也是计算机系统的内核与基石

CPU 提供两种类型的指令集
  • 特权指令集
    • 普通程序不能直接调用
    • 普通程序可以调用操作系统提供的接口，去间接调用特权指令
  • 普通指令集

操作系统会将 CPU 提供的特权指令集封装为系统调用（System Call，syscall）

组成

内核

驱动程序

接口库

外围

作用

抹平不同计算机硬件之间的差异


功能（通用目的）

操作系统位于底层硬件与用户之间，是两者沟通的桥梁。
  • 用户可以通过操作系统的用户界面，输入命令。
  • 操作系统可以对命令进行解释，驱动硬件设备，实现用户要求。

进程管理

（Processing management）

CPU 时间片切割

不管是常驻程序或者应用程序，他们都以进程为标准运行单位。

进程管理指的是操作系统调整多个进程的功能。

除了进程管理之外，操作系统还有负担起进程间通信（IPC）、进程异常终止处理、死锁（Dead Lock）侦测及处理等较为艰深的问题。

结构

内核

kernel

操作系统最核心的部分，通常运行在最高特权级，负责提供基础性、结构性的功能。

接口库

共享对象

是一系列特殊的程序库，他们的职责在于把系统所提供的基本服务包装成应用程序所能使用的编程接口（API），是最靠近应用程序的部分。

驱动程序

是最底层的、直接控制和监视各类硬件的部分，他们的职责是隐藏硬件的具体细节，并向其他部分提供一个抽象的、通用的接口。

外围

所谓外围，是指操作系统中除了以上三类之外的其他部分，通常是用于提供特定高级服务的部件。

例如，在微内核结构中，大部分系统服务，以及 UNIX/Linux 中各种守护进程都通常被划归此列。

  • 系统调用：system call
    • Linux 大约提供了 300 多个系统调用指令
  • 库调用：library call

others

程序运行格式：
  • Windows: EXE，dll (dynamic link library)
  • Linux: ELF, so (shared object)

程序：指令 + 数据
  • 指令：只读
  • 数据：读写，指令加工的对象

程序：算法 + 数据结构


库调用，系统调用：都是允许被调用的程序


编程语言：
  • 汇编语言：微码编程
    • 系统中某些与硬件相关的特有代码
    • 驱动程序
  • 高级语言：C、C++
    • 系统级应用：MySQL、OS、驱动程序
  • 高级语言：Java、Python
    • 应用程序




分类

操作系统的种类相当多，各种设备安装的操作系统可从简单到复杂，可分为智能卡操作系统、实时操作系统、传感器节点操作系统、嵌入式操作系统、个人计算机操作系统、多处理器操作系统、网络操作系统和大型机操作系统。[4] 

应用领域

可分为桌面操作系统、服务器操作系统、嵌入式操作系统；

所支持用户数

可分为单用户操作系统(如MSDOS、OS/2．Windows)、多用户操作系统(如UNIX、Linux、MVS)；

源码开放程度

可分为开源操作系统(如Linux、FreeBSD)和闭源操作系统(如Mac OS X、Windows)；

硬件结构

可分为网络操作系统(Netware、Windows NT、OS/2 warp)、多媒体操作系统(Amiga)、和分布式操作系统等；

操作系统环境

可分为批处理操作系统(如MVX、DOS/VSE)、分时操作系统( 如Linux、UNIX、XENIX、Mac OS X)、实时操作系统(如iEMX、VRTX、RTOS，RT WINDOWS)；

存储器寻址宽

可以将操作系统分为8位、16位、32位、64位、128位的操作系统。早期的操作系统一般只支持8位和16位存储器寻指宽度，现代的操作系统如Linux和Windows 7都支持32位和64位。
